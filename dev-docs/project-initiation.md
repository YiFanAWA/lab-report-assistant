# 实验报告助手｜立项兼容入口

本文档是 Sliver 启动门禁的兼容入口。项目真实立项内容以 [project-charter.md](project-charter.md) 为准；这里不维护第二份产品真相。

## 1. 项目定义

项目名称为“实验报告助手”。它是面向高校本科生的实验材料采集、实验要求拆解、数据分析执行与 Word/PPT 交付物生成工作流。

项目边界、目标用户、典型场景、核心问题和第一版目标均以 [project-charter.md](project-charter.md) 的已确认锁定版本为准。

## 2. 想法收敛

用户的原始想法是：让用户爬取论文或特定网站，并根据实验要求自动生成 PPT 或实验报告。

已收敛为“单一推荐主线”：不做黑盒一键生成器，而做“证据化实验工作流”。系统把实验要求、公开资料、证据卡片、数据分析、受控 Python 执行、实验大纲、Word/PPT 生成拆成有边界、可检查、可回退的工作区。

潜在杠杆点：把普通 AI 代写从“文本生成”提升为“资料证据 + 真实执行结果 + 可审阅交付物”的闭环。

## 3. 目标

第一版目标以 [project-charter.md](project-charter.md) 第 2 章和第 8 章为准：至少跑通一个典型数据分析实验，从实验项目创建到 Word/PPT 下载。

## 4. 非目标

第一版不支持所有网站、不绕过登录或付费限制、不做手机 App/小程序/多人协作/自由拖拽工作流、不做完整论文复现、不生成虚构实验结果、不把教学分析包装为医疗诊断。

完整排除项以 [project-charter.md](project-charter.md) 第 3.2 和 3.3 节为准。

## 5. 第一闭环

第一闭环是：

```text
创建实验项目
-> 输入课题和实验要求
-> 生成结构化任务单
-> 添加公开资料和数据文件
-> 生成证据卡片和数据概览
-> 生成并确认分析方案
-> 受控执行 Python
-> 确认真实结果
-> 生成统一大纲
-> 生成并下载 Word 与 PPT
```
