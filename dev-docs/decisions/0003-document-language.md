# ADR 0003｜项目文档必须使用中文

- 日期：2026-06-16
- 状态：已确认
- 决策人：项目负责人
- 关联真源：`AGENTS.md`、`dev-docs/README.md`

## 背景

项目负责人明确要求：所有文档文件都需要中文。

本项目当前仍处于架构与开发计划阶段，文档是真源。为了减少后续 agent 漂移、误读和中英混写，项目自有文档必须统一中文写作。

## 决策

项目自有文档必须使用中文，包括：

- 根目录 `AGENTS.md`。
- `dev-docs/` 下的立项、架构、验收、实施计划和决策记录。
- 后续新增的 `docs/` 用户文档、发布说明、运行说明和交接文档。

允许保留必要的技术专名、文件路径、代码标识符、命令、协议名、产品名和文件格式名，例如 `Python`、`React`、`FastAPI`、`CSV`、`Word`、`PPT`、`.docx`、`dev-docs/project-charter.md`。

## 约束

- 不得使用英文段落或英文标题作为项目自有文档正文。
- 后续新增文档必须先中文命名、中文解释，再写路径或技术标识符。
- 若引用外部英文规范，只能摘录必要标识符，并用中文说明其含义。
- `skills/` 下下载的第三方 skill/plugin 仓库源码属于外部 vendored 资料，不在本次强制改写范围内；如需改写，必须另行决策，避免破坏上游内容。

## 验收

- `dev-docs/README.md` 索引本决策。
- 当前项目自有文档中的英文标题和英文叙述已改为中文。
- 后续文档审查时，把英文段落和英文标题视为文档缺陷。
