# ADR 0002｜进入架构与开发计划阶段

- 日期：2026-06-16
- 状态：已确认
- 决策人：项目负责人
- 关联真源：`dev-docs/project-charter.md`

## 背景

项目负责人已确认立项文档，并明确批准进入架构与开发计划阶段。

同时，项目负责人明确约束：本阶段仍然不写业务代码、不安装依赖、不初始化技术框架。

## 决策

项目进入“架构与开发计划阶段”。本阶段允许创建和维护以下内部真源：

- `dev-docs/architecture.md`
- `dev-docs/acceptance.md`
- `dev-docs/implementation-plan.md`
- 与阶段、技术选型和执行批准相关的 ADR

## 约束

- 所有架构和计划必须以 `dev-docs/project-charter.md` 为依据。
- 不得扩大 MVP 范围。
- 技术栈可以推荐，但不得执行初始化。
- 进入代码阶段前必须获得项目负责人新的明确批准。

## 验收

- 架构文档说明单一主线架构、唯一归属层和禁止路径。
- 验收文档说明阶段门禁、证据要求和漂移锁。
- 实施计划说明未来代码阶段任务，但不创建代码。
