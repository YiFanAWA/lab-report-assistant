# 决策 0007：代码阶段正式开始执行

> 日期：2026-06-16
> 状态：已接受
> 决策人：项目负责人
> 关联文档：[project-charter.md](../project-charter.md)、决策 0006、[specs/0001-project-workspace-and-scaffold.md](../specs/0001-project-workspace-and-scaffold.md)

## 背景

项目已完成立项、技术栈、项目规范名、V1 登录边界、首个演示课题、样例数据来源、大模型供应商和依赖版本复核的全部前置确认。决策 0006 已创建代码阶段批准记录。

项目负责人明确说"确定可以写了"，批准进入代码实现阶段。

## 决策

本项目代码阶段正式启动。当前执行范围严格限定在 SPEC 0001 第一开发切片：项目工作区与脚手架。

在本文档创建后的所有工作中，agent 可以创建业务代码、安装依赖、初始化技术框架、编写测试和运行验收命令。

## 约束

以下约束在代码阶段全程有效：

- 只实现 SPEC 0001 定义的范围，不越界实现后续切片内容。
- 不接入 DeepSeek 真实调用。
- 不上传或读取样例数据文件。
- 不做注册登录。
- 不让前端拥有业务状态机。
- 不提交真实 API 密钥。
- 不把数据库错误原文直接暴露给 API 响应。
- 所有实现必须遵守 `AGENTS.md`、`architecture.md`、`tech-stack.md` 中的约束。
- 每个任务完成后必须提供运行命令和验证证据。

## 执行范围

本决策授权执行 `implementation-plan.md` 中的：

- 任务 1：仓库与框架脚手架（全部）。
- 任务 2：核心合同（项目工作区最小子集）。
- 任务 3：项目工作区核心（项目创建、查询和持久化）。

## 停止条件

第一切片完成后必须暂停，由项目负责人确认后再进入下一切片。

## 影响

- 当前切片完成后，需要更新 `dev-docs/README.md`、`dev-docs/acceptance.md` 的证据记录和 `implementation-plan.md` 中任务 1 至任务 3 最小子集的勾选状态。
- 后续切片仍需要单独创建 SPEC 文档。

## 重新评估条件

出现以下情况时必须更新本决策或新增决策记录：

- 用户要求停止代码阶段；
- SPEC 0001 范围需要调整；
- 实现发现已有架构或技术栈约束不正确；
- 本地环境无法满足框架的最小运行要求。
