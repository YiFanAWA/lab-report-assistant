# ADR 0001｜锁定项目立项文档为后续真源

- 日期：2026-06-16
- 状态：已确认
- 决策人：项目负责人
- 关联真源：`dev-docs/project-charter.md`

## 背景

项目负责人已确认《实验报告助手｜项目立项文档》中的项目定义、第一版目标、功能范围、排除项、用户路径、核心业务对象、初步技术路线、里程碑和验收标准。

当前项目仍处于立项确认阶段。在项目负责人明确批准进入下一阶段之前，不创建业务代码、不安装依赖、不初始化技术框架。

## 决策

从本决策生效起，后续所有产品设计、任务拆解、技术选型、代码实现和验收工作，都必须以 `dev-docs/project-charter.md` 为依据。

任何项目方向、功能边界或验收标准变化，必须先更新立项文档和决策记录，再调整开发计划。

## 约束

- 不得擅自扩大 MVP 范围。
- 不得偏离已确认的产品主线。
- 不得只在对话中口头修改项目方向。
- 不得在未获批准时进入业务代码、依赖安装或技术框架初始化。

## 验收

- `dev-docs/project-charter.md` 存在并标记为已确认锁定。
- `dev-docs/README.md` 将 `project-charter.md` 列为当前真源。
- 本决策记录存在并说明后续变更流程。
